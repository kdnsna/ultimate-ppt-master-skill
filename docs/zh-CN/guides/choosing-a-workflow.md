# 选择交付路线

先按最终要交付的文件选择路线，不需要先理解内部引擎。

## 可编辑 PowerPoint

适用于经营复盘、咨询方案、培训、政务金融和需要他人继续修改的正式材料。

- 主要产物：`.pptx`。
- 保留文字、形状、图表、表格和备注等可支持的原生对象。
- 结构预览只用于确认故事，不等同于最终文件。
- 对外交付前在目标 PowerPoint/WPS 字体环境复核。

## Web Deck

适用于发布会、公开演讲、Demo Day 和浏览器优先的视觉叙事。

- 主要产物：独立 HTML。
- 重点检查桌面、390px 移动端、键盘、滚轮和触控浏览。
- 来源页、事实截止日期和推断边界必须保留。

## 双交付

当同一份资料既要线上展示又要正式文件时使用。

- PPTX 与 Web Deck 共用资料、故事板和 `slideId`。
- 两种输出分别生成和复核，不把网页截图直接当成可编辑 PPTX。
- 每种产物都有自己的质量状态和下载项。

## 已有 PPTX

已有文件需要检查时先用 [PPTLint](https://github.com/kdnsna/pptlint)。需要修改时走 preservation-first 路线，只处理明确页面和对象；不要把旧文件混入新建演示流程后整套重做。

## 从任务到下载

1. 在工作台输入任务并添加真实资料。
2. 修改并确认故事板的标题、结论和结构方案。
3. 创建本地项目；Bridge 允许时启动 Codex，否则复制命令运行。
4. 工作台每 3 秒发现 `exports/`、`ppt/` 与质量报告白名单中的真实产物。
5. 待复核文件可下载；只有产物 `passed` 且全部页面批准后才标记已交付。

如果没有真实资料，系统会保留证据缺口并阻止把结构稿包装成正式成品。英文完整说明见 [Choosing a Workflow](../../guides/choosing-a-workflow.md)。
