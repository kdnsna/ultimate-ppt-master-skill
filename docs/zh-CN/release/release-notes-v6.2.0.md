# v6.2.0 发布边界：生成闭环

> **未发布草案。** 本文件定义拟发布、可独立回滚的 v6.2.0 边界；只有里程碑提交合并且 CI 通过后，才能创建 tag 和 GitHub Release。

拟发布的 v6.2.0 只处理故事板生产合同和本地产物闭环：

- `DeckSession → DeckIR` 保留页面顺序、`slideId`、标题、结论、角色和已选结构；
- 统一支持 4–24 页，无资料时继续阻断正式生产；
- Bridge 提供受限的白名单产物列表、以附件响应下载产物，以及安全 Agent 启动；
- 工作台每 3 秒轮询真实产物，页面隐藏时暂停，刷新后按 `projectPath` 恢复；
- 结构稿明确标识，`passed` 产物与全页批准共同构成交付门禁。

回滚边界：Web V6 工作区、Bridge 产物 API 与 DeckIR 转换可作为一组回滚，不依赖 v6.3 的公开案例和中文入口。
