# v6.3.4 候选说明：会话归属与真实交付

> **未发布候选。** 这是 v6.3 强化工作中记录的回滚切片。它出现在 `main` 或 Pages 上，不证明已有独立 v6.3.4 tag、GitHub Release 或 marketplace 发布。

[English candidate notes](../../release/release-notes-v6.3.4.md)

## 白话更新栏

- 每个 handoff 都拥有会话标识；SSE 订阅与工作台只接受属于当前会话的进度。
- 同标题并发项目使用不同目录，不再共用仅依赖时间戳的路径。
- 产物发现只能在已验证 handoff 内进行，且只扫描白名单输出/报告位置。
- 报告只是辅助证据，不是最终成品。交付必须同时拥有真实 PPTX、Web Deck、PDF 或压缩包，对应验证为 `passed`，且每页均已批准。
- artifact 列表/下载、Agent 启动与单页修订共用 manifest HMAC 身份校验，不存在一个接口信任、另一个接口拒绝的项目。

## 候选边界

本切片只负责会话级进度、唯一项目身份、已验证产物发现与最终成品交付谓词。资料真实性属于 v6.3.3，用户可见精修体验属于 v6.3.5。

## 发布前证据

- 并发与交叉会话 SSE 测试，包括双标签页；
- 大量同标题项目并发创建，不得覆盖文件；
- 仅报告、零字节、未完成和超出白名单的产物不得满足交付；
- Agent 启动与下载均拒绝未通过共享身份边界的项目。

## 独立回滚边界

将会话过滤、唯一目录创建、产物发现元数据和最终交付谓词作为一个整体回滚。保留 v6.3.3 的 source id/provenance 和 v6.3.2 项目认证；回滚时绝不删除已创建的项目目录。

版本化发布前，应将本单元保留为独立提交或明确的 revert map，并复核持久化字段兼容性。这份源码说明是逻辑边界，不是可独立定位的发布产物。
