# Tech Trend Web Deck Quality Checklist

## Source Integrity

- [ ] Every public signal has a source URL or citation note.
- [ ] Dates are included for recent announcements, releases, or comparisons.
- [ ] Star counts and popularity signals are described as directional evidence.
- [ ] Facts, interpretation, and recommendations are visually separated.

## Story

- [ ] The cover states a trend thesis, not only a topic name.
- [ ] The first three slides work without narration.
- [ ] The deck explains why now, what changed, and what the audience should do.
- [ ] Caveats and risks are not hidden at the end only.

## Visual And Delivery

- [ ] The Web Deck uses one style system consistently.
- [ ] Dense evidence pages alternate with calmer thesis pages.
- [ ] Screenshots or images show actual product, workflow, or state when available.
- [ ] Desktop and mobile previews have no text or media overlap.
- [ ] Optional PPTX appendix keeps summary, sources, and speaker notes editable.
