# Tech Trend Web Deck Source Skeleton

## Topic

Local-first AI presentation workflows for builders and AI tool users.

## Audience

Developers, product builders, AI tool users, and technical decision makers.

## Trend Thesis

The next useful AI presentation workflow is not a single black-box generator. It is a local-first chain: public or private source material, Markdown handoff, local agent tools, editable outputs, and visible quality gates.

## Public Signals To Provide

- GitHub repositories, launch pages, changelogs, or docs with dates.
- Public product announcements or ecosystem updates.
- Evidence of adoption, such as stars, forks, release cadence, community examples, or integration patterns.
- Screenshots or diagrams that can be safely shown in public.

## Suggested Signals

- Markdown conversion tools for agent input.
- MCP or agent tool interoperability.
- Markdown-first slide tools.
- Editable PPTX generation libraries.
- Open-source AI presentation products.

## Narrative Arc

1. The old way: upload files, get a pretty but hard-to-edit deck.
2. The shift: agents need inspectable sources and local tools.
3. The new chain: source -> Markdown -> handoff -> agent production -> QA.
4. What builders should do: design for reuse, visibility, and local trust.
5. What users get: better control, better editability, fewer black-box surprises.

## Required Output

- Primary: magazine-style HTML Web Deck.
- Optional: editable PPTX summary appendix.

## Notes For The Agent

- Use public sources only.
- Put source dates near claims when recency matters.
- Do not overstate adoption from star counts alone.
- Check desktop and mobile readability before delivery.
