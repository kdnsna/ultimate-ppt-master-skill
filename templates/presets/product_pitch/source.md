# Product Pitch

## Audience
Investors, partners, early customers, and product teams.

## Product One-Liner
Ultimate PPT Master turns real source material into editable PowerPoint decks and cinematic Web Decks through a local Agent workflow.

## User Pain
Teams can generate plausible AI slides quickly, but the output often becomes a screenshot-like artifact that is hard to edit, hard to verify, and disconnected from source files.

## Why Now

- Local coding agents can read files, run converters, inspect outputs, and repair generated assets.
- Teams increasingly want AI help without uploading sensitive business material.
- Presentation workflows need both formal handoff files and high-impact web presentations.

## Target User

- Founder preparing a product demo or investor update.
- Consultant building a client proposal from source documents.
- Business owner preparing executive review material.
- Internal team turning training or research material into reusable decks.

## Solution

The product works as a local production hub:

1. Web Experience collects source material and route choices.
2. Local Bridge stages a handoff project on the user's machine.
3. Agent Skill reads the project, converts files, plans the story, generates output, previews, checks, and repairs.
4. The user receives editable PPTX, Web Deck, or both.

## Proof Points

- Supports PDF, DOCX, XLSX, PPTX, URL, Markdown, and pasted text.
- Preserves editable PPTX as the formal delivery route.
- Adds magazine and Swiss Style Web Deck routes for demos and public sharing.
- Keeps source material local by default.

## Demo Flow

1. Add a real source file or paste a brief.
2. Choose product pitch preset and output route.
3. Download or send the handoff kit to local Bridge.
4. Ask Codex or another local Agent to execute `agent-prompt.md`.
5. Inspect final files and quality checklist.

## Roadmap Signal

- v2.4: content and template presets.
- v2.5: template gallery with screenshots.
- v2.6: preset output quality benchmark.

## Ask

Use the product pitch preset to create a Web Deck first, then produce an optional editable PPTX appendix for handoff.

## Constraints

- Show real workflow states rather than abstract AI imagery.
- Keep the first three slides readable without narration.
- Mark assumptions clearly when traction or market data is synthetic.
